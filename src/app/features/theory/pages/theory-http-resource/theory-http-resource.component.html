L'API Resource : Gérer les données asynchrones avec les signaux

Dans les versions récentes d'Angular (depuis Angular 19), la nouvelle fonction rxResource et l'API resource simplifient considérablement la gestion des requêtes HTTP et de l'état asynchrone par rapport à l'utilisation classique de HttpClient et des observables seuls.

Cette API s'intègre parfaitement avec les Signals et fournit nativement l'état de la requête sans avoir besoin de gérer manuellement des variables comme isLoading, error, ou de devoir se désabonner (unsubscribe).

Une ressource expose notamment les signaux suivants :

  • value() : Les données récupérées par la requête (pouvant être undefined si non résolues).
  • isLoading() : Un booléen indiquant si la requête est en cours de chargement.
  • error() : L'erreur éventuelle retournée par la requête.
  • hasValue() : True si une valeur a été chargée avec succès.
  • status() : Le statut métier plus détaillé de la ressource (Idle, Loading, Error, Resolved, Reloading, etc.).

Utilisation avec rxResource et HttpClient

Pour utiliser la puissance des ressources avec vos services HttpClient (qui retournent des observables), Angular propose rxResource situé dans @angular/core/rxjs-interop. Cette fonction permet d'appeler httpClient.get() tout en bénéficiant de la nouvelle API de signal.

Service

  • request : Récupère ou suit des paramètres sous forme réactive. Lorsque ces paramètres changent, rxResource annule la requête précédente (s'il y en a une en cours) et en lance une nouvelle.
  • loader : Effectue la requête (ici via HttpClient) en recevant l'objet params, qui contient notamment la propriété request (la valeur renvoyée par la fonction request précédente).

Composant (Template)

Dans le template, plus besoin d'utiliser le pipe async, de souscrire manuellement ni de gérer des variables booléennes de chargement : on lit directement l'état de notre ressource !

>

Rechargement forcé (Reload)

Il est très facile de forcer la ressource à refaire un appel réseau sans pour autant changer les paramètres de la requête. La ressource expose une méthode reload() à cet effet.

En appelant reload(), la ressource se met à jour, fait repasser l'état de chargement à true et ré-exécute le loader HTTP sur le réseau.
