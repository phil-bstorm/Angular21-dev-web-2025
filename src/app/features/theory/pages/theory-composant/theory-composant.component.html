Un composant est un élément fondamental d'une application Angular. Il est responsable de l'affichage d'une partie de l'interface utilisateur et de la gestion des interactions avec l'utilisateur.

Tout est composant dans Angular. Une application Angular est une hiérarchie de composants imbriqués les uns dans les autres.

Si on analyse rapidement la structure de ce projet, on peut constater que le projet est séparé en fonctionalité (theory, demo,...), chaque fonctionalité est encore découper en composant (theory-binding, theory-composant, theory-cycle-de-vie,...) et composant qui peut encore être découper selon son fonctionnement.

Un composant est un regroupement de 3 fichiers:

  • Un fichier TypeScript (.ts) qui contient la logique du composant.
  • Un fichier HTML (.html) qui contient le template du composant.
  • Un fichier CSS (.css) qui contient les styles du composant.

Comment créer un composant?

Pour créer un composant, il suffit de lancer la commande suivante :
ng generate component nom-du-composant
Ou en version courte :
ng g c nom-du-composant

⚠️ Le composant se crée à l'endroit où vous vous trouvez dans le terminal.
Il est donc préférable de:

  • soit se déplacer dans le dossier du projet (avec la commande cd src/app) avant de lancer la commande de création de composant.
  • soit de renseigné le chemin où on veut créer le composant.
    ⚠️ même si vous préciser le chemin, ce chemin démarre quand-même d'où vous vous trouvez dans le terminal
    Par exemple pour créer un composant login dans le dossier src/app/auth:
    ng g c auth/login

Astuce: pour éviter d'avoir des problèmes avec la création de composant, il est recommandé de se déplacer avec cd dans le dossier src/app (cd src/app).
Ainsi, vous pourrez créer vos composants en précisant le chemin à partir de src/app et en bénéficiant de l'autocomplétion (TAB).

Ce qui aura pour effet de créer un dossier contenant les fichiers suivants :

  • nom-du-composant.component.html : le template HTML du composant.
  • nom-du-composant.component.ts : la logique du composant.
  • nom-du-composant.component.scss : le fichier de styles du composant.

🔎️Note: si un dossier avec le nom du composant existe déjà, les fichiers seront simplement rajotuer au dossier.

Comme résultat de la commande dans le terminal, celui-ci nous indeque que 3 fichiers ont été créés (.html, .ts, .scss).

selector

Le selector est une propriété du décorateur @Component qui permet de définir le nom de la balise HTML qui sera utilisée pour afficher le composant.

Par défaut, le selector est le nom de la classe du composant en kebab-case (séparé par des tirets).

Par exemple, si le composant s'appelle hello, le selector sera app-hello.

Le selector peut être utilisé de différentes manières pour afficher le composant dans le template du composant parent.

Par exemple, pour afficher le composant Hello dans le template du composant parent, on peut utiliser les syntaxes suivantes :

Balise

Si j'utilise selector: 'app-hello'

Class CSS

Si j'utilise selector: '.appHello'

Attribut

Si j'utilise selector: '[appHello]'

Combinaison

Il est possible d'utiliser plusieurs sélecteurs pour un même composant. Par exemple, si on souhaite afficher le composant Hello avec les sélecteurs app-hello, .appHello et [appHello], on peut définir le selector comme suit :
selector: 'app-hello,.appHello,[appHello]'

🔎 chaque sélecteur est séparé par une virgule

View encapsulation

Par défaut, l'encapsulation est définie sur « Emulated ». Ça signifie que le style défini dans un composant ne pourra pas modifier celui de ses parents. Si l'on souhaite changer ce comportement, il faut modifier la propriété encapsulation.
