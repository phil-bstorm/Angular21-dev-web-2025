Contexte:

Dans Angular, chaque composant traverse une série d'étapes de cycle de vie, de sa création à sa destruction. Angular propose des hooks spécifiques pour intercepter ces étapes et exécuter du code en conséquence.

Objectif:

Maîtriser les hooks de cycle de vie pour répondre à différents besoins, tels que l'initialisation du composant, la gestion des changements ou encore la libération de ressources avant la destruction du composant.

Qu'est-ce qu'un hook de cycle de vie ?

Un hook de cycle de vie est une méthode fournie par Angular que vous pouvez implémenter dans vos composants pour exécuter du code à des moments précis du cycle de vie d'un composant. Voici un aperçu des différents hooks disponibles :

1. Le hook ngOnChanges

Quand est-il appelé ?

Ce hook est appelé à chaque fois qu'une propriété liée au composant change.

Utilité : Il permet de réagir aux changements de valeurs des entrées (inputs) d’un composant.

🔍 Ce hook reçoit un objet SimpleChanges qui contient les anciennes et nouvelles valeurs des propriétés liées.

2. Le hook ngOnInit

Quand est-il appelé ?

Ce hook est appelé une seule fois, immédiatement après la première liaison des données au composant.

Utilité : Utilisé pour effectuer des initialisations comme la récupération de données ou la configuration d'un état initial.

3. Le hook ngDoCheck

Quand est-il appelé ?

Ce hook est appelé à chaque cycle de détection des changements d'Angular.

Utilité : Il permet de détecter les changements manuellement, par exemple, si vous avez besoin d'une logique plus fine pour vérifier certains changements non détectés par Angular.

4. Le hook ngAfterContentInit

Quand est-il appelé ?

Ce hook est appelé une seule fois après qu'Angular ait projeté le contenu externe dans le composant.

Utilité : Il est utilisé lorsque vous devez interagir avec le contenu projeté dans un composant (contenu d'un autre composant inséré via ng-content)

5. Le hook ngAfterContentChecked

Quand est-il appelé ?

Ce hook est appelé après chaque vérification du contenu projeté dans le composant.

Utilité : Utilisé pour effectuer des actions après chaque cycle de vérification du contenu projeté dans le composant.

6. Le hook ngOnDestroy

Quand est-il appelé ?

Ce hook est appelé juste avant que le composant soit détruit, par exemple lors de la navigation vers une autre page.

Utilité : Utilisé pour nettoyer les ressources (désabonnement aux observables, arrêt de timers, etc.).
