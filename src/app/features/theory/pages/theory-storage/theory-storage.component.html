Types de stockages

Lorsque l'on veut stocker des donnés qui persistent, on peut utiliser le localStorage ou le sessionStorage ou le cookie.

LocalStorage

Le localStorage est un stockage persistant, c'est-à-dire que les données restent disponibles même après la fermeture du navigateur. Il est accessible via l'objet localStorage et permet de stocker des paires clé-valeur.

SessionStorage

Le sessionStorage est un stockage temporaire, c'est-à-dire que les données sont supprimées lorsque l'onglet ou la fenêtre du navigateur est fermée. Il est accessible via l'objet sessionStorage et permet également de stocker des paires clé-valeur.
Étant donner que le SessionStorage se vide lorsque l'onglet est fermé il n'est pas utilisé en Angular.

Cookie

Un cookie est un petit fichier texte qui est stocké sur l'ordinateur de l'utilisateur. Il est envoyé au serveur à chaque requête HTTP. Les cookies sont souvent utilisés pour stocker des informations de session ou des préférences utilisateur.
Note si on souhaite utiliser les cookies, il faut un package supplémentaire: ngx-cookie-service

Sauvegarder des données

Pour sauvegarder des données dans le localStorage ou le sessionStorage, on utilise la méthode setItem(key, value). La clé est une chaîne de caractères qui identifie la donnée, et la valeur est la donnée à stocker.
Note: il faut convertir les objets en JSON avant de les stocker à l'aide de JSON.stringify.

Récupérer les données

Pour récupérer des données dans le localStorage ou le sessionStorage, on utilise la méthode getItem(key). La clé est la même que celle utilisée lors de la sauvegarde.
Note: il faut convertir les données JSON en objet à l'aide de JSON.parse.
